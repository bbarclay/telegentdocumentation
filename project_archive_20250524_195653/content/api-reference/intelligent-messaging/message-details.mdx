## ğŸ“ˆ Message Status Values

| Status | Description |
|--------|-------------|
| `Queued` | Message is queued for delivery |
| `Sent` | Message has been sent to the carrier |
| `Delivered` | Message was successfully delivered |
| `Failed` | Message delivery failed |
| `Undelivered` | Message could not be delivered |

## ğŸ’» Code Examples

### Python

```python
import requests

url = "https://api.telegent.com/v1.0/message/details"

params = {
    "MessageId": "TMID-948AFE64-A7A9-45D4-A30A-DB7CFB57A2B6"
}

headers = {
    "Authorization": "Bearer YOUR_API_TOKEN"
}

response = requests.get(url, headers=headers, params=params)
data = response.json()

print(f"Message ID: {data['MessageId']}")
print(f"Status: {data['Status']}")
print(f"Cost: ${data['Cost']}")
print(f"Direction: {data['Direction']}")
print(f"Message Type: {data['MessageType']}")
```

### JavaScript

```javascript
const getMessageDetails = async (messageId) => {
  const response = await fetch(
    `https://api.telegent.com/v1.0/message/details?MessageId=${messageId}`,
    {
      method: 'GET',
      headers: {
        'Authorization': 'Bearer YOUR_API_TOKEN'
      }
    }
  );

  const data = await response.json();
  
  console.log('Message Details:', {
    id: data.MessageId,
    status: data.Status,
    cost: data.Cost,
    type: data.MessageType
  });
  
  return data;
};

// Usage
getMessageDetails('TMID-948AFE64-A7A9-45D4-A30A-DB7CFB57A2B6');
```

- **ğŸ“Š Status Tracking**: Use this endpoint to track the status of a specific message
- **ğŸ’° Cost Information**: The `Cost` field provides the actual cost incurred for sending the message
- **ğŸ–¼ï¸ MMS Content**: For MMS messages, media content URLs or base64 data will be included in the response
- **â° Retention Period**: Message details are typically retained for a limited period (e.g., 30-90 days)
- **ğŸ”§ Troubleshooting**: This endpoint is useful for confirming delivery, checking message status, and troubleshooting failed messages
- **ğŸ” Alternative Search**: To search for messages by phone number instead of message ID, use the `/message/search` endpoint
- **âŒ Not Found**: If a message is not found, verify that the MessageId is correct and that the message was sent within the retention period
