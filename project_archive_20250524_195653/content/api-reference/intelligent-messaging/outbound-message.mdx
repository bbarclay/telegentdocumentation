## üìä Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `MessageId` | string | Unique identifier for the message |
| `Direction` | string | Direction of the message (always "OUTBOUND" or "Outbound") |
| `Type` | string | Type of message (p2p or a2p) |
| `CampaignId` | string | Campaign identifier if provided |
| `Cost` | number | Cost of sending the message |
| `Status` | string | Current status of the message (typically "Queued") |
| `To` | array | Recipients of the message |
| `From` | string | Sender's phone number |
| `Owner` | string | Phone number that owns the conversation |
| `Body` | string | Text content of the message |
| `Country`/`Region` | string | Country or region of the recipient |
| `CreatedOn` | string | Timestamp when the message was created |
| `MessageType` | string | Type of message (sms or mms) |
| `MmsMedia` | array | Media URLs for MMS messages |
| `MmsBase64` | array | Base64-encoded media for MMS messages |

## üíª Code Examples

### Python

```python
import requests
import json

url = "https://api.telegent.com/v1.0/message/outbound"

# SMS Example
payload = {
    "To": [
        {
            "Number": "+12016093801"
        }
    ],
    "From": "+18018018011",
    "Owner": "+18018018011",
    "Body": "Hello there!!!"
}

headers = {
    "Content-Type": "application/json",
    "Authorization": "Bearer YOUR_API_TOKEN"
}

response = requests.post(url, headers=headers, json=payload)
data = response.json()

print(f"Message ID: {data['MessageId']}")
print(f"Status: {data['Status']}")
print(f"Cost: ${data['Cost']}")
```

### JavaScript


## üìù Usage Notes

- **üìû Phone Number Format**: The `From` number must be a valid phone number in your account with messaging enabled
- **üåç E.164 Format**: All phone numbers should be in E.164 format (e.g., +12125551234)
- **üë§ Owner Field**: The `Owner` field establishes message thread ownership and is typically the same as the `From` number
- **üì± SMS vs MMS**: For SMS messages, only include the `Body` parameter. For MMS, add `MmsMedia` URLs, `MmsBase64` content, or both
- **üéØ Campaign Tracking**: The `CampaignId` parameter is optional but useful for tracking campaigns and message groups
- **üìä Status Updates**: The `Status` field indicates initial status. Updates may be delivered to your callback URL
- **üí∞ Cost Tracking**: The `Cost` field indicates the cost of sending the message
- **üë• Multiple Recipients**: Send to multiple recipients by adding more objects to the `To` array
- **üîÑ Individual Messages**: When sending to multiple recipients, a separate message is generated for each recipient
